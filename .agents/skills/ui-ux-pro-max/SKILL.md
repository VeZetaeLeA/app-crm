# UI/UX Pro Max Skill

This skill provides advanced design intelligence for UI and UX tasks, from building new components to auditing existing screens for accessibility and visual consistency.

## When to Use This Skill
Use this skill when the user:
- Asks to "design a new dashboard page"
- Requests "a review of my UI for accessibility"
- Needs to "create a new UI component" (buttons, tables, modals)
- Asks "how to make my site look more modern"
- Needs "font pairings" or "color schemes" for a new brand
- Requires "data visualization" (charts, graphs)
- Asks for "UX recommendations" on a specific user flow

## How to Apply Design Intelligence

### 1. Database of Design Styles (50+)
The skill includes a vast range of styles:
- **Minimalism**: Clean, white space, focused on content.
- **Glassmorphism**: Translucent surfaces with blur (current system default).
- **Brutalism**: Bold, raw, high-contrast, often unconventional.
- **Neumorphism**: Soft shadows and highlights, "extruded" surfaces.
- **Bento Grid**: Modular, grid-based layout for dashboards.

### 2. Color Palettes (160+)
Tailored to product types:
- **Enterprise**: Zinc, Indigo, Slate.
- **Creative**: Fuchsia, Cyan, Amber.
- **Health**: Emerald, Teal, Sky.
- **Fintech**: Gold, Navy, Rose.

### 3. Font Pairings (50+)
Using Google Fonts:
- **Precise**: Sora (Headings) + Outfit (Body).
- **Legible**: Inter + Roboto.
- **Modern**: Montserrat + Open Sans.

### 4. UX Guidelines (90+)
Covers common flows and components:
- **Input Fields**: Labels, error messages, focus states.
- **Navigation**: Information architecture, mobile menus.
- **Onboarding**: Progressive disclosure, feedback cycles.

### 5. Accessibility (WCAG)
The skill ensures compliance with accessibility standards (WCAG 2.1). Always check for:
- **Contrast Ratios**: Minimum 4.5:1 for normal text.
- **Semantic HTML**: Proper use of `<h1>`, `<button>`, `<nav>`.
- **Keyboard Navigation**: Focus indicators and tab order.

## Task-Specific Workflows

### UI Review / Audit
1. Capture screenshots of the UI.
2. Analyze the visual hierarchy and consistency.
3. Check contrast and accessibility.
4. List specific recommendations with rationale.

### Building New Features
1. Understand the goal and user persona.
2. Propose 2-3 style options with visual descriptions.
3. Design the layout using the Bento Grid or Bento Box pattern.
4. Detail the component states (hover, focus, disabled).
